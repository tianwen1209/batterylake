Documentation
Technical reference for using the BatteryLake Benchmark datasets and evaluation framework.

Getting Started

Clone the repository and install the conda environment:

git clone https://github.com/tianwen1209/BatteryLake-Benchmark-DataPrep.git
cd BatteryLake-Benchmark-DataPrep
conda create -n batterylake python=3.10
conda activate batterylake
pip install -r requirements.txt

Dataset Registry

All datasets are tracked in dataset_registry.csv with columns: dataset_id, dataset_name, data_category, doi, source_url, assigned_student, ref_name, status, metadata_done, timeseries_done, cycle_summary_done, dataset_note_done, qc_done, last_updated, notes.

Running ETL

Each dataset has a dedicated ETL script in scripts/ following the naming pattern etl_dataset_XX.py. Run with:

python scripts/etl_dataset_01.py --input raw/dataset_01/ --output processed/dataset_01/

Evaluation

The benchmark evaluation framework uses evaluate.py and dataset_interface.py to run all five baseline models across selected datasets and split protocols.

About

BatteryLake Benchmark is a comprehensive benchmark platform for lithium-ion battery prognostics and health management (PHM). The project curates, standardizes, and evaluates battery cycling datasets from research labs worldwide, enabling fair comparison of machine learning models for state-of-health estimation and remaining useful life prediction.

Physical Plausibility Checks
High-signal checks that determine whether a dataset can safely enter preprocessing and benchmark training.
5 passed · 1 warning
  • Voltage Range Validation
    All cell voltages within 2.0V–4.5V nominal operating range for the stated chemistry.
  • Energy Balance Check
    Charge/discharge energy integral consistency; coulombic efficiency remains within 95–105% per cycle.
  • Capacity Monotonicity
    Degradation trajectory follows expected non-increasing trend with allowable recovery windows.
  • Temperature Consistency
    Cell surface temperature must remain within ±3°C of stated test condition; one dataset needs review.
  • Timestamp Integrity
    Monotonically increasing timestamps with no negative intervals or unreasonable gaps above 24h.
  • Current Direction Consistency
    Charge and discharge current signs follow one convention throughout the dataset.
